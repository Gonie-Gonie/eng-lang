# v0.4-preview Release Notes

`v0.4-preview` promotes the read-only I/O slice of the general programming and
side-effect policy. It keeps EngLang's core execution native and reviewable
while allowing small companion text/config files to participate in workflows
with source hash provenance.

EngLang remains preview software. Language syntax, runtime behavior, and
artifact schemas are not stable yet.

```text
- source hash provenance in review.json, result.engres, and report_spec.json
- official/11_read_only_io mini example
- side-effect policy docs updated for GP-2 implemented scope
```

## Read-Only I/O Scope

The supported preview forms are:

```eng partial
notes_text = read text args.notes
json_text = read json args.config_json
toml_text = read toml args.config_toml

notes_text_2 = read_text(args.notes)
json_text_2 = read_json(args.config_json)
toml_text_2 = read_toml(args.config_toml)
```

The values are strings. `read json` and `read toml` intentionally do not yet
produce structured EngLang objects. Structured JSON/TOML promotion remains
future data-boundary work.

## Provenance

Read-only I/O records environment dependency metadata:

```text
kind
expression
resolved_value
status
source_hash
```

`source_hash` is present when the file can be read. Missing sources are still
reported with a missing status and null hash.

## Official Example

Run:

```bat
eng.exe check examples\official\11_read_only_io\main.eng --review
eng.exe run examples\official\11_read_only_io\main.eng --save-artifacts
```

The example combines:

```text
- typed CSV promote
- TextFile/JsonFile/TomlFile args
- read-only text/json/toml inputs
- unit-aware statistics and print
- explicit summary CSV export
```

## Deferred

```text
- structured JSON/TOML values
- write text/json
- summary CSV overwrite hardening
- output manifest for generated artifacts
- copy/move/delete
- external process and network effects
```

Those are tracked in the general programming and side-effect policy. The
immediate next target is `v0.5-preview`: write/export hardening plus an output
manifest.
