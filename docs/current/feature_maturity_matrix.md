## Solver Vocabulary

Use these labels consistently in public docs:

| Term | Current Meaning |
|---|---|
| One-state fixed-step runner | The narrow runtime path used by official one-state thermal examples. |
| One-state fixed-step thermal workflow | The supported shape for `02_simple_system` and measured-vs-simulated simulation output. |
| One-state adaptive Heun thermal workflow | The supported `solver = adaptive_heun` shape for one-state thermal simulation with a fixed output TimeGrid and adaptive internal substeps. |
| Two-state source-equation fixed-step ODE workflow | The supported `der(state)` source-equation shape used by `examples/official/20_multi_state_thermal`, with scalar or Time-indexed TimeSeries inputs and explicit Euler/RK4 fixed-step execution. |
| Source-equation RHS evaluator | The runtime evaluator that lowers supported arithmetic `C * der(state) eq rhs` source equations into canonical derivative values for fixed-step ODE solving. |
| Adaptive Heun state-space workflow | The internal `solver = adaptive_heun` shape for continuous state-space `der(x) eq A * x + B * u` simulations with a fixed output TimeGrid and adaptive internal substeps. |
| State-space fixed-step runner | The internal vector runtime path used by state-space examples, including multi-state Euler/RK4 and discrete A/B execution. |
| Solver metadata | Review/result/report-spec metadata describing recognized equations, boundaries, methods, and limitations. |
| Solver plan | Planning metadata for equation ordering and future numeric work; not proof of a general solver. |
| Connection constraint consistency check | Runtime consistency artifact for generated component connection constraints; not a physical component-graph solve. |
| Thermal component boundary assembly | The supported constrained component graph shape used by `examples/official/23_thermal_component_assembly` and `examples/official/24_linear_algebraic_thermal_node`: Thermal connection sets, literal boundary seeds, simple component-local equations, and square dense linear residual solves. |
| General solver | Planned only. Do not present the supported one-state thermal or two-state source-equation workflows as a broad nonlinear/DAE/adaptive/component solver. |
| Component graph solver | Supported only for the constrained Thermal boundary assembly shape above; broader component graph solving and production multi-domain simulation remain planned. |

## Status Terms

| Status | Meaning |
|---|---|
| Stable | Public behavior covered by the 1.0.0 stable-core scope and breaking-change policy. |
| Supported | Usable, documented, tested, and visible through diagnostics or artifacts for the stated scope, but not covered by stable breaking-change policy. |
| Internal | May have code, tests, examples, or artifacts on `main`, but is not a public-supported release feature. |
| Planned | Intended future work with no supported implementation contract yet. |

## Completion Policy

A feature is not complete merely because an example passes. A feature is
complete only when its language rule, parser/AST support, semantic/type/unit
check, runtime/check behavior, diagnostics, IDE metadata or inspector support,
report/review artifact, official example or internal fixture, tests, and
documentation are aligned for the stated scope.
