# v0.7-preview Release Notes

`v0.7-preview` promotes structured runtime messages. The goal is to keep quick
CLI feedback simple while giving IDEs, CI scripts, and reviewers a
machine-readable message artifact.

## Version Labels

Public label:

```text
v0.7-preview
```

Workspace package version:

```text
0.7.0-preview
```

## What Changed

- `print "..."` remains direct CLI/debug output.
- New `log <level> "..."` statements support `debug`, `info`, `warn`, and
  `error`.
- `warn "..."` is intentionally not a separate command; use `log warn "..."`.
- `log` interpolation is type- and unit-aware like `print`.
- Invalid or missing log levels produce `E-LOG-LEVEL-001`.
- `review.json` records print/log metadata with a `level` field.
- `eng run` prefixes structured log messages as `[level] message`.
- Saved runs write `build/result/run_log.json`.
- `output_manifest.json` records the saved `run_log` artifact.
- The native IDE artifact panel exposes run-log paths and in-memory payloads.
- New official mini example:
  `examples/official/14_run_log/main.eng`.

## Example

```eng partial
Q_design: HeatRate [kW] = 12.5 kW
E_day: Energy [kWh] = 42 kWh

print "case ready"
log info "Q design = {Q_design: .1 kW}"
log debug "daily energy raw = {E_day: .2 kWh}"
log warn "review high load case"
log error "operator acknowledgement required"
```

## Run Log Artifact

Saved runs write:

```text
build/result/run_log.json
```

The artifact uses:

```text
format = eng-run-log-v1
runtime_version
source_path
message_count
messages:
  index
  level
  message
  line
```

`print` records use level `print`. Structured `log` records use the declared
level.

## Boundary

Runtime messages are not durable engineering result surfaces. Use:

```text
print       quick direct CLI/debug output
log level   structured runtime message metadata
export      reproducible scalar CSV artifacts
write       explicit generated text/json outputs
report      reviewable engineering report artifacts
```

Project-wide display-unit policy remains out of scope for this slice.

- test/assert/golden support
