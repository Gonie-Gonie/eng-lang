# v0.3-preview Release Notes

Status: ready as the third public preview package after local release checks.

This release keeps the language and artifact formats in preview. It promotes
the syntax/dataflow unification slice and implements the first side-effect
policy seed: typed path helpers with provenance-visible `exists` checks.

## Scope

`v0.3-preview` is a user-testable preview package for typed engineering data
workflows, native IDE inspection, path-oriented workflow inputs, and
reproducible report/review artifacts.

Included:

```text
- eng.exe doctor/check/run/build/view
- top-level execution as the default root workflow
- root args block, importable const, pure scalar fn, and relative file imports
- parenthesis-light command syntax for built-in workflow verbs
- where local calculation blocks and with option/display blocks
- unit-aware print interpolation
- explicit summary CSV export under build/result
- typed path defaults and pure path helpers
- provenance-visible exists checks in review/result/report-spec artifacts
- typed CSV promote through official examples
- unit-aware TimeSeries calculations and statistics
- PlotSpec v1, SVG output, report.html, review.json, and report_spec.json
- basic standalone package execution
- native tester IDE
- experimental eng-lsp.exe smoke/snapshot path
- packaged VS Code extension preview
- curated user PDF guide and language grammar PDF
```

New or promoted in this release:

```text
- file/dir path defaults for args
- join/parent/stem/extension path helpers
- exists path checks typed as Bool
- environment_dependencies in review.json
- provenance.environment_dependencies in result.engres and report_spec.json
- official/10_path_policy mini example
```

This is preview software:

```text
The language and artifact formats are not stable.
Broad read/write filesystem I/O, filesystem mutation, process execution, and
network access are not public-supported runtime features yet.
```
