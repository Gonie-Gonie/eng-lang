# v0.4-preview Release Notes

Status: ready as bytecode VM and result v1 preview.

## Scope

v0.4-preview turns the previous artifact skeleton into an executable native path:

```text
.eng -> check -> entry selection -> .engbc v1 -> VM seed -> result.engres v1
```

Included:

```text
- workspace version bumped to 0.4.0
- script signature metadata in AST
- entry point table in semantic output
- entry selection helper
- `eng entries <file.eng>`
- `eng run <file.eng> --entry <name>`
- `eng build <file.eng> --entry <name>`
- E-ENTRY-NOT-FOUND-001
- E-ENTRY-MULTIPLE-001
- bytecode v1 text format
- bytecode encode/decode
- bytecode object records for scalar/table/array values
- bytecode instructions for entry, load, and result write
- native VM seed
- VM object store summary
- result.engres v1 typed payload
- bytecode hash provenance
- entry metadata in review.json/report.html/result.engres
- missing entry error example
```

## Commands

```bat
.\dev.bat setup
.\dev.bat doctor
.\dev.bat ci
.\dev.bat package
```

Useful manual checks:

```bat
target\debug\eng.exe entries examples\04_plotting\main.eng
target\debug\eng.exe run examples\04_plotting\main.eng --entry main
target\debug\eng.exe view build\result\result.engres
target\debug\eng.exe run examples\05_error_messages\missing_entry.eng
```

The missing-entry command should fail with `E-ENTRY-NOT-FOUND-001`.
