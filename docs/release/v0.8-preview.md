# v0.8-preview Release Notes

`v0.8-preview` promotes explicit external process execution. The goal is to let
EngLang workflows call external tools without hiding side effects: process
intent is visible in review metadata, and runtime output is captured as a
machine-readable artifact.

## Version Labels

Public label:

```text
v0.8-preview
```

Workspace package version:

```text
0.8.0-preview
```

## What Changed

- New `result = run command "tool"` syntax captures an external process as a
  typed `ProcessResult`.
- `run command` must bind a result so exit status and output stay reviewable.
- Supported process options are `args`, `cwd`, and `allow_failure`.
- Non-zero process exits fail the run by default.
- `with { allow_failure = true }` records expected failures as process results.
- `review.json` records `process_runs[]` metadata.
- Saved runs write `build/result/process_results.json`.
- `output_manifest.json` records the saved `process_results` artifact.
- The native IDE artifact panel exposes process-result paths and in-memory
  payloads.
- New official mini example:
  `examples/official/15_process_result/main.eng`.

## Example

```eng partial
process_result = run command "cmd"
with {
    args = ["/C", "echo", "eng-process-ok"]
}

log info "process result captured"
```

## Process Results Artifact

Saved runs write:

```text
build/result/process_results.json
```

The artifact uses:

```text
format = eng-process-results-v1
runtime_version
source_path
process_count
processes:
  binding
  command
  args
  cwd
  exit_code
  success
  stdout
  stderr
  duration_ms
  status
  line
```

## Boundary

External process execution is now explicit and reviewable, but stable process
sandboxing is not claimed yet. Use `run command` for carefully scoped workflow
steps, not as a hidden general-purpose scripting escape hatch.

Runtime surfaces remain separated:

```text
print          quick direct CLI/debug output
log level      structured runtime message metadata
run command    explicit process execution and ProcessResult metadata
export         reproducible scalar CSV artifacts
write          explicit generated text/json outputs
report         reviewable engineering report artifacts
```

## Still Deferred

- network/download/cache effects
- broad filesystem mutation outside generated-output boundaries
- stable safe/normal/repro profile enforcement
- stable process sandboxing
- first-class Summary objects
- project-wide display-unit policy
- test/assert/golden support

## Next Target

The immediate next target is `v0.9-preview`: test blocks, assertions, and
golden-style checks for examples and artifacts.
