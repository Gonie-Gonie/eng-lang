# v1.0.0 Release Notes

`v1.0.0` is the first stable-core EngLang package. It does not claim that every
track is complete. It stabilizes the documented data-to-report workflow,
artifact family, packaged runner, and native tester path while keeping broader
solver, editor-service, modeling, domain, and runtime-optimization work outside
the stable contract.

```

```

## Stable Core

- Top-level workflow execution with root `args { ... }`.
- Fast `=` declarations, explicit quantity declarations, unit/quantity
  diagnostics, and `:=` rejection.
- Typed CSV promotion for the official data boundary.
- Supported HeatRate TimeSeries statistics and trapezoidal integration path.
- Measured-vs-simulated official workflow with typed measured/weather CSV
  TimeSeries, one-state fixed-step simulation output as `sim.T_zone`, RMSE
  metric, validation result, time-alignment metadata, and multi-series PlotSpec.
- PlotSpec v1 line and multi-series line plot, SVG, report HTML, report spec,
  result, review, run log, process results, test results, and output manifest
  artifacts.
- Unit-aware `print`, structured `log <level>`, explicit summary CSV export,
  write outputs, process results, and local test/assert/golden checks within the
  documented boundaries.
- `eng run --profile safe|normal|repro` basics for runtime side-effect policy:
  safe rejects explicit workflow side effects, normal is default, and repro
  records profile diagnostics in artifacts.
- Standalone package with `.engpkg`, `.lock`, dependency copy, Args help, and
  `run.bat`.
- Tauri tester IDE smoke path for the stable workflow.

## Supported/Internal/Planned Outside Stable Core

- General nonlinear, adaptive, or multi-state system solving.
- Broad table/TimeSeries expression execution outside the documented path.
- Uncertainty and data-driven modeling engines.
- LSP/VS Code as a persistent editor contract.
- Native JIT/AOT execution or speedup claims.
- Domain/component package ecosystem and numeric multi-domain solving.
- Broad filesystem/network side effects and full process sandboxing.
- Workspace-wide test discovery and filtering.

## Policy

The stable scope is documented in:

```text
docs/current/stable_core_scope.md
docs/reference/breaking_change_policy.md
```

Stable artifact contracts may gain optional fields in `1.x`, but removing or
changing required stable fields requires a major version unless the release
notes identify a severe correctness or data-loss fix.
