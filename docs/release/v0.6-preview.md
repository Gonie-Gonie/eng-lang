# v0.6-preview Release Notes

`v0.6-preview` promotes the constrained filesystem mutation slice of the
general programming track. It keeps EngLang's side-effect philosophy intact:
operations that touch files must be explicit, reviewable, and recorded in
runtime artifacts.

## Version Labels

Public label:

```text
v0.6-preview
```

Workspace package version:

```text
0.6.0-preview
```

## What Changed

- `copy <source> to <destination>` top-level workflow statements.
- `move <source> to <destination>` top-level workflow statements.
- `delete <target>` top-level workflow statements.
- `move` and `delete` require `with { confirm = true }`.
- `delete dir(...)` also requires `with { recursive = true }`.
- copy/move/delete effects are constrained to generated-output boundaries in
  the current preview.
- `review.json` now includes `file_operations[]`.
- `output_manifest.json` records `copy_file`, `move_file`, and `delete_file`
  artifact entries.
- CLI output reports file operation paths with an `fs:` line.
- The native IDE artifact panel shows file operation paths.
- New official mini example:
  `examples/official/13_file_operations/main.eng`.

## Example

```eng partial
copy file("data/template.txt") to "ops/copied_note.txt"

move "ops/copied_note.txt" to "ops/archive/copied_note.txt"
with {
    confirm = true
    overwrite = true
}

write text "ops/scratch.txt", "temporary generated note"

delete "ops/scratch.txt"
with {
    confirm = true
}
```

## Safety Boundary

The current preview is intentionally conservative:

```text
- copy can copy a source-relative UTF-8 text file or generated output file into build/result
- move operates on generated output paths under build/result
- delete operates on generated output paths under build/result
- absolute paths and .. traversal are rejected for generated output targets
- process and network effects remain deferred
```

This is enough for reviewable generated-file workflows without opening broad
project filesystem mutation.

## Review And Manifest Records

`eng check --review` exposes:

```text
file_operations[]
  operation
  source
  destination
  line
```

`eng run --save-artifacts` writes:

```text
build/result/output_manifest.json
```

with entries such as:

```text
copy_file
move_file
delete_file
```

## Still Deferred

- broad filesystem mutation outside generated-output boundaries
- `mkdir` and directory listing
- process execution and `ProcessResult`
- network/download/cache effects
- stable safe/normal/repro profile enforcement
- test/assert/golden support

## Next Target

The immediate next target is `v0.7-preview`: `log`/`warn` syntax, structured
runtime message metadata, and a run log artifact that distinguishes debug
output from reproducible artifacts.
