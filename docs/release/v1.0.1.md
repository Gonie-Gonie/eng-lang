# v1.0.1 Release Notes

Planned scope: documentation/release-contract hardening.

This maintenance slice keeps the `v1.0.0` stable-core contract intact while
tightening public wording around Stable, Supported, Internal, and Planned
tracks.

## Stable

- Reaffirm `v1.0.0` as the current stable-core public line.
- Keep the stable contract limited to typed CSV promote, unit-aware TimeSeries
  workflows, TimeSeries statistics/integration metadata, PlotSpec/SVG,
  report/review artifacts, measured-vs-simulated artifacts, packaged runner
  smoke, and native tester workflow.

## Supported

- Clarify the system/equation path as a one-state fixed-step thermal workflow
  with solver metadata and solver-plan artifacts.
- Keep measured-vs-simulated simulation output scoped to the official one-state
  fixed-step runner and typed TimeSeries artifacts.

## Internal

- Keep state-space under `examples/internal/18_state_space_metadata`: it now
  produces a one-state TimeSeries trajectory, but remains internal until the
  supported state-space workflow is complete.
- Keep domain/component graph metadata framed as connection metadata and a
  connection constraint consistency check, not as numeric multi-domain solving.
- Keep uncertainty, data-driven modeling, LSP/VS Code, and JIT/AOT tracks
  outside the stable-core contract.

## Planned

- General nonlinear solvers, DAE solvers, adaptive or multi-state solvers,
  numeric component graph solving, domain package registries, class/domain
  object runtime lowering, full LSP, JIT/AOT speedup claims, and broad
  ANN/data-driven modeling frameworks remain outside the stable-core contract.

- `.\dev.bat release-check`
