# v0.9-preview Release Notes

`v0.9-preview` promotes local workflow verification. The goal is to let examples
and user projects carry lightweight checks beside the calculation: typed
assertions for scalar values and golden comparisons for generated artifacts.

## Version Labels

Public label:

```text
v0.9-preview
```

Workspace package version:

```text
0.9.0-preview
```

## What Changed

- New named `test "name" { ... }` blocks run during `eng run`.
- `assert` is valid only inside a `test` block.
- Assertions support `==`, `!=`, `>`, `>=`, `<`, and `<=`.
- `within` adds a unit-aware tolerance for equality-style checks.
- `golden "artifact" matches file("expected")` compares generated outputs under
  `build/result` with source-relative expected files.
- `review.json` records `tests[]` metadata.
- Saved runs write `build/result/test_results.json`.
- `output_manifest.json` records the saved `test_results` artifact.
- The native IDE artifact panel exposes test-result paths and in-memory
  payloads.
- New official mini example:
  `examples/official/16_test_assert_golden/main.eng`.

## Example

```eng partial
test "summary values" {
    assert mean_Q > 0 kW
    assert E_coil == 1.26 kWh within 0.02 kWh
    golden "summary.csv" matches file("golden/summary.csv")
}
```

## Test Results Artifact

Saved runs write:

```text
build/result/test_results.json
```

The artifact uses:

```text
format = eng-test-results-v1
runtime_version
source_path
test_count
failed_count
tests:
  name
  status
  line
  assertions
  goldens
```

## Boundary

This is local source-file verification, not a full workspace test runner.
Workspace discovery, test filtering, fixtures, and stable CI profiles remain
deferred. `print`, `log`, `export`, `write`, `run command`, and `test` keep
separate purposes:

```text
print          quick direct CLI/debug output
log level      structured runtime message metadata
export/write   reproducible generated artifacts
run command    explicit process execution and ProcessResult metadata
test           runtime assertions and golden artifact comparison
report         reviewable engineering report artifacts
```

metadata, `test_results.json`, and output manifest records.

## Still Deferred

- workspace-wide test discovery and filtering
- network/download/cache effects
- broad filesystem mutation outside generated-output boundaries
- stable safe/normal/repro profile enforcement
- stable process sandboxing
- first-class Summary objects
- project-wide display-unit policy

## Next Target

The immediate next target is `v1.0`: stable core hardening. The stable label
should be used only after supported preview features are either promoted,
documented as preview, or explicitly deferred.
