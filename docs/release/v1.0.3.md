# v1.0.3 Release Notes

Status: draft. Do not publish this release until the native IDE and curated user
documentation have been manually reviewed on a clean Windows test PC.

## Scope

v1.0.3 is the next post-v1.0 hardening release. It replaces the first tester IDE
surface with a more conventional native editor workflow and cleans up the
release documentation bundle.

Planned and implemented in this preparation slice:

```text
- workspace version prepared as 1.0.3
- native IDE starts on the integrated HVAC user-test example
- VS Code-like layout: Explorer, Code, Result, Inspector sidebar, Problems/Output/Artifacts
- compact Windows-native font stack uses Segoe UI for UI text and Cascadia/Consolas fallback for code
- Settings controls light/dark theme, UI density, UI/code font sizes, window presets,
  soft wrapping, and panel default sizes, saved under build/ide/settings.json
- Explorer supports opening .eng files from examples, stdlib, and tutorial sources
- Explorer shows `examples/official` before compatibility regression,
  diagnostic, and data-quality fixtures
- Explorer supports creating new .eng files from a starter template
- editor uses EngLang syntax highlighting
- editor code pane has independent vertical and horizontal scrolling
- editor horizontal scrolling appears only when long source lines exceed the visible pane
- editor auto-closes parentheses, brackets, braces, and quotes
- diagnostics are surfaced as toolbar counts, Problems rows, and line background tinting
- completions suggest current-file variables, source identifiers, keywords, quantities, units, and snippets
- first completion suggestion is visible below the code pane and applies with Tab
- completions are separated from symbol inspection to avoid right-panel overlap
- Inspector shows variables with quantity kind, display/canonical unit,
  dimension, source, expression, and unit derivation path
- Inspector shows schema columns, constraints, missing policies, and CSV
  promotion summaries
- Runtime Summary tab inspects result.engres inside the IDE after Run
- Runtime Summary shows uncertainty distribution/method/percentiles and ML
  train/test metrics, coefficients, leakage status, and loss history
- Result pane is inside the main work area, resizable, independently scrollable,
  and no longer competes with the right Inspector sidebar
- Run preview renders PlotSpec points directly inside the native IDE with
  grid lines, x/y ticks, zero baseline, and distinct line/scatter/bar/histogram rendering
- Artifacts tab shows report, report_spec, PlotSpec, manifest, SVG, result, review, and bytecode paths
- package docs are curated PDF assets rather than a full copy of developer markdown files
- dev.bat setup installs repo-local portable Python and oodocs requirements for document generation
- PowerShell PDF generation remains as a no-extra-tool fallback
- current-status layer documents supported/preview/experimental/planned scope
- feature maturity matrix prevents seed-only examples from being treated as release support
- LLM_CONTEXT.md and docs/llm/load_map.yml define the minimal context path for future agents
- master-plan/current.md points at the current v9 plan; historical plans are left to git history
- `°C` is supported as a user-facing alias for canonical `degC` AbsoluteTemperature spelling
- examples/README.md defines the official, compatibility regression,
  diagnostic, data-quality, and scratch-file roles
```

## Portable Package

The package command should create:

```text
dist\englang-preview-v1.0.3-windows-x64.zip
dist\englang-preview-v1.0.3-windows-x64.zip.sha256
dist\englang-user-test-guide-v1.0.3.pdf
dist\release-manifest.txt
```

The unpacked package should contain:

```text
eng.exe
eng-ide.exe
eng-lsp.exe
docs\EngLang_User_Test_Guide.pdf
examples\
stdlib\
tools\vscode-englang\
tools\englang-vscode-preview-1.0.3.vsix
README.txt
```

The package must not include the full developer markdown documentation tree.

## Release Gate

Before tagging `v1.0.3`:

```bat
.\dev.bat setup
.\dev.bat ci
.\dev.bat docs-check
.\dev.bat ide-check
.\dev.bat artifacts-check
.\dev.bat package-smoke
```

Manual IDE review must confirm:

```text
- no extra console window in the packaged release build
- eng-lsp.exe --smoke runs from the unpacked portable package
- visible readable text, buttons, panels, and tabs
- no overlap in Inspector/Completions/Runtime sidebar or bottom Problems/Artifacts panels
- Code and Result are visible as side-by-side main panes and scroll independently
- dragging the Code/Result divider resizes the panes without hiding the right sidebar
- Result pane does not cover or push the right Inspector/Completions/Runtime sidebar
- Code pane does not show horizontal scrolling for normal-width official examples
- typing a variable/keyword/unit prefix shows a suggestion and Tab applies it
- typing `(`, `[`, `{`, `'`, or `"` auto-inserts the matching closer
- Explorer opens official examples and creates a scratch .eng file
- Explorer lists Official Examples before regression and fixture folders
- Check updates lint diagnostics without running the program
- Run generates report and plot artifacts
- Plot preview appears inside the IDE for official plotting examples with readable grid/ticks
- Runtime Summary shows uncertainty details for official/04_uncertainty_core
- Runtime Summary shows ML metrics and coefficients for official/05_data_driven_modeling
- Inspector shows schema, CSV, and unit-path details for official/03_integrated_hvac
- Report and Plot SVG buttons open generated files
- README and docs index point first to current status and feature maturity
- v1.1/v1.2 are described as experimental/planned, not release-supported
- core path wording says Python is optional documentation tooling only
- `24 °C` and schema units such as `[°C]` are accepted and converted as AbsoluteTemperature
```

## Known Limits

The native IDE is still an integrated tester IDE, not a full LSP environment.
The VS Code extension remains the secondary editor path. v1.1 feature work
should not begin until the v1.0.3 user-test IDE and documentation gate is clean.
